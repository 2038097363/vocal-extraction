# 视频人声提取工具

这是一个 Windows 本地桌面工具。它会先从视频中提取音频，再用 Demucs 模型分离人声，最后输出为你选择的音频格式。

## 快速使用

1. 第一次使用双击 `Install.bat`，安装本地环境并下载模型。
2. 之后双击 `启动工具.vbs` 打开工具，不会弹出黑色命令行窗口。
3. 选择视频文件、输出目录、输出格式和模型。
4. 点击“开始提取人声”。

如果需要把人声中的文字也提取出来，勾选“同时识别人声中的文字，输出 TXT 和 SRT”。工具会在输出目录里生成同名 `.txt` 文本和 `.srt` 字幕文件。

已经部署好的内容会放在当前文件夹：

- `.conda-env`：本地 Python/FFmpeg/Demucs 环境
- `models`：Demucs 模型权重
- `cache`：安装缓存和运行缓存
- `outputs`：默认输出目录

## 推荐设置

日常默认推荐：

- 输出格式：`FLAC - 无损压缩`
- 分离模型：`htdemucs`
- 采样率：`保持模型默认`
- 文字识别：需要文稿或字幕时再勾选

想要更高分离质量：

- 分离模型改为 `htdemucs_ft`
- 输出格式仍建议选 `FLAC` 或 `WAV`

`htdemucs_ft` 会比默认模型慢一些，因为它会组合多个模型结果。

## 文字识别

文字识别使用本地 faster-whisper 模型，默认模型是 `base`。它会优先识别分离后的人声轨，通常比直接识别原视频混音更干净。

输出文件：

- `.txt`：纯文本转写
- `.srt`：带时间轴的字幕文件

识别模型越大通常越准，但也越慢。默认 `base` 更轻量，第一次部署和日常使用更稳；需要更高准确率时可以改选 `small` 或 `medium`。

## 格式怎么选

简单记法：

- `FLAC`：最推荐。无损、体积比 WAV 小，适合保存和后期。
- `WAV`：无损且最直接，适合专业音频软件继续处理，但体积最大。
- `MP3`：兼容性最好，适合发给别人或普通播放。
- `M4A/AAC`：手机和剪辑软件常用，同体积下通常比 MP3 更好。
- `OGG`：开放格式，适合网页、游戏或开源工作流。
- `OPUS`：语音效率高，体积小，但部分软件不一定支持。

更完整的说明见 `docs\格式与音质说明.txt` 或 `docs\FORMAT_GUIDE.md`。
文字识别说明见 `docs\文字识别说明.txt`。

## 关于音质

工具内部会先生成 `32-bit float WAV` 临时音频，避免中间步骤进行有损压缩。

但需要注意：如果原视频只有一条混音音轨，人声分离是 AI 根据混音推断出来的人声，不等于从视频里无损取出一条原本独立存在的人声轨。选择 `FLAC` 或 `WAV` 可以避免最终输出阶段继续损失音质，但不能消除 AI 分离本身可能带来的伴奏残留或轻微处理痕迹。

## 支持格式

输入视频格式由 FFmpeg 支持，常见的 `mp4`、`mkv`、`mov`、`avi`、`webm`、`flv`、`wmv`、`ts` 等都可以处理。

输出格式：

- FLAC
- WAV
- MP3
- M4A/AAC
- OGG/Vorbis
- OPUS

## 启动方式

推荐双击 `启动工具.vbs`。它会在后台调用 `Run.bat`，不会显示 CMD 黑窗。

`Run.bat` 保留给排查问题时使用。

## 命令行用法

也可以使用命令行：

```bat
CLI.bat "D:\video\demo.mp4" -o "D:\audio" -f flac
```

可选输出格式：`flac`、`wav`、`mp3`、`m4a`、`ogg`、`opus`。

## 重新安装或修复

`Install.bat` 可以重复运行。模型下载脚本会校验权重文件，如果下载中断或文件损坏，重复运行 `Install.bat` 即可补齐。
